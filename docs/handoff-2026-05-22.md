## Enforcement Layer Architecture (for context)

| Layer | Type | Owns |
|-------|------|------|
| `CLAUDE.md` | Advisory | Behavioral norms, workflow philosophy |
| Hooks | Deterministic structural | Per-tool-call enforcement (dangerous cmds, scope validation) |
| Reviewer/Opponent | Semantic | Spec compliance, scope drift |
| CI | Deterministic gate | File size, forbidden patterns, secrets |

All hooks exit 0 (WARN tier — never blocks). Contract hook warns when writes fall outside declared scope.

## What's Next

The architecture is complete. From memory, two remaining frontiers:

1. **Adoption** — use the system on real projects and surface friction. The tooling (mb init, mb validate, mb doctor, install scripts) is ready; the gap is actual use revealing edge cases.

2. **Semantic identity detection** — the system detects file-level degeneration (compaction depth, missing lineage roots) but not concept-level drift: duplicate decisions, contradictory claims, stale superseded entries. Detection-first when approached — flag for human review, no auto-merge until semantic certainty exists.

No active plans for either. Start a new brainstorm session when ready.
