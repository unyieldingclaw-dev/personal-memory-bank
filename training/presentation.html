Memory Bank Standard

Persistent Context for AI Coding Assistants

Enterprise Training Guide

The Problem

Every AI coding session starts like this:

You: "Add a new API endpoint"

AI: "What's your tech stack?"

You: "FastAPI, PostgreSQL, React..."

AI: "What's your architecture?"

You: "Microservices with adapters..."

AI: "What are your constraints?"

You: *sighs* "Must work without Docker..."

⏱️ 15 minutes of context setup... every. single. time.

The Solution: Memory Bank

You: "Add a new API endpoint"

AI: "I'll create a FastAPI endpoint following your multi-adapter pattern, with PostgreSQL support and SQLite fallback per your techContext. Adding to port 8004 based on your service allocation."

✨ Immediate productive work. Zero context setup.

How? AI reads structured files at the start of every session.

The 5 Memory Bank Files

File Purpose Update Frequency
projectbrief.md Requirements, constraints, goals Rarely
systemPatterns.md Architecture, code patterns, anti-patterns When patterns change
techContext.md Tech stack, ports, environment When tech changes
activeContext.md Current focus, next steps Every session
progress.md Done, in progress, planned After milestones

Handoff: Seamless Across Sessions

Memory-bank reads automatically. Handoff bridges sessions when context fills up. You never manually re-explain.

The seamless loop

Session 1
  AI reads memory-bank/ automatically (no action needed)
  You work. AI tracks decisions, patterns, progress.
  Context gets full.
  You type: "Handoff"
  AI writes handoff.md with session state, then stops.

Session 2 (fresh chat)
  AI reads memory-bank/ automatically (still no action needed)
  AI sees handoff.md, reads it, merges into memory-bank files.
  AI deletes handoff.md.
  AI continues exactly where you left off.

Threshold by IDE

IDE Handoff at Why
Claude Code 65% Auto-compacts at ~75% — handoff fires before that so nothing is lost
Cursor 80% Rules re-inject every response; 80% is safe

What you do: Type "Handoff" when the context bar gets close to the threshold. That's it. Memory-bank reading is already automatic.

Security Guardrails

3-Tier Protection System

Tier Behavior Examples
BLOCK AI refuses, no override Commit secrets, force push main
CONFIRM AI pauses for approval Delete files, amend commits
WARN AI notes risk, proceeds Large changes, missing tests

Result: Trust that AI won't accidentally break things.

Code Quality Standards

✅ AI Will Do

  • Run tests before saying "done"
  • Check for lint errors
  • Add WHY comments for complex logic
  • Handle error cases
  • Prefer editing over creating files

❌ AI Won't Do

  • Add obvious comments
  • Leave commented-out code
  • Swallow exceptions silently
  • Claim "done" without verification
  • Create unnecessary files

Accessibility (WCAG 2.1 Level AA)

UI code only. Auto-applies via glob-scoped Cursor rule. On-demand audit via /accessibility-review.

Always-on rule

.cursor/rules/accessibility.mdc activates only on:

.html .htm .jsx .tsx .vue .svelte .astro .css .scss .sass .less

Backend and CLI projects never see accessibility rules — zero noise.

On-demand audit

/accessibility-review scans a diff, file, or folder against the nine dimensions below and emits a severity-rated findings table + remediation checklist.

Usage: /accessibility-review src/components/Form.tsx

Nine dimensions (all hard requirements)

Area Key requirement
Semantic HTML Native elements for their purpose; logical heading hierarchy; landmark regions.
ARIA Only when native HTML is insufficient; never override native semantics; aria-live for dynamic updates.
Keyboard navigation Every interactive element reachable; logical tab order; focus trap in modals; skip-nav links.
Focus indicators Never remove default outline without a visible replacement (≥3:1 contrast).
Color & contrast 4.5:1 normal text; 3:1 large text and UI; never convey information by color alone.
Forms & inputs Programmatic labels (never placeholder); text error messages tied via aria-describedby.
Images & media Descriptive alt for informative, alt="" for decorative; captions for video, transcripts for audio.
Motion & animation Respect prefers-reduced-motion; no content flashing more than 3×/second.
Testing NVDA / JAWS / VoiceOver verification; accessibility tree inspection; don't rely solely on automated tools.

Full standard: standards/ACCESSIBILITY.md

Structured Logging

Queryable logs. No secrets. Correlation across services.

Why it matters

  • Incident response — filter by user_id, order_id, correlation_id in one query.
  • SRE / on-call — JSON logs feed straight into aggregators.
  • Compliance — PII auto-redacted before it hits disk (emails, phones, SSN).
  • Noise reduction — LOG_SKIP_PATHS drops /health, /ready by default.

Required pattern

# GOOD — structured, queryable logger.info("order_created", order_id="ORD-123", total=149.99) # BAD — unstructured string logger.info(f"Order {id} done") # correlation ID logger = logger.bind( correlation_id=req_id)

Python: structlog  ·  TypeScript: pino  ·  Env: LOG_LEVEL, LOG_FORMAT=json, LOG_SKIP_PATHS, LOG_CORRELATION_HEADER

Feature Workflow

Prevents the #1 AI failure mode: writing code before understanding the problem.

Brainstorm -> Spec -> Plan -> Implement (TDD) -> Simplify -> Security Review -> Commit

Each phase

  • Brainstorm — explore, propose 2–3 approaches.
  • Spec — write validated design to docs/specs/.
  • Plan — bite-sized plan, exact file paths.
  • Implement — TDD: failing test → code → green.
  • Simplify — clarity review, no behavior change.
  • Security Review — scan for 9 patterns.
  • Commit — descriptive message; push.

Skip rule

Jump straight to Implement for:

  • Single-file fixes
  • Typos
  • Config changes
  • Changes under 20 lines

How to trigger

Claude Code: /feature-dev runs the full 7-phase flow.

Cursor: .cursor/rules/workflow.mdc enforces it automatically.

Multi-agent /code-review

Three role-separated reviewers. Uncorrelated contexts. No cross-bias.

Step 3 — 3 parallel subagents

Each sees only the code and its own lens.

  • 🔐 Security — secrets, injection, auth, crypto, eval, sensitive-data logs.
  • ⚡ Performance — N+1, unbounded loops, blocking I/O, pagination, caching.
  • 🎨 Style & Standards — length, nesting, naming, dead code, duplication.

Severity: CRITICAL / HIGH / MEDIUM / LOW

Step 4 — Test Coverage Review

Main agent checks happy path, edge cases, error paths. If tests are missing, it generates them — one file per changed module.

Step 5 — Opponent Auditor

Final subagent sees all findings and confirms / downgrades / rejects each one, and surfaces anything the three reviewers missed.

Verdict column: ✓ confirmed · ↓ downgraded · ✗ false positive

Usage: /code-review (git diff)  ·  /code-review src/auth/login.py  ·  /code-review src/api/

In Cursor: "do a code review" / "review src/api/routes.py".

5-Minute Setup

What it does: adds a memory-bank/ folder to your project with five files the AI reads at the start of every chat — so it already knows your project, every time.

Steps

  1. Open PowerShell (Windows) or Terminal (Mac).
  2. Move into your project folder:

    Windows

    cd C:\path\to\your-project

    Mac

    cd /path/to/your-project
  3. Paste the matching command, press Enter:

    Windows

    irm https://gitlab.com/tmobile/ere/memory-bank/-/raw/master/scripts/init-memory-bank.ps1 | iex

    Mac

    curl -sSL https://gitlab.com/tmobile/ere/memory-bank/-/raw/master/scripts/init-memory-bank.sh | bash
  4. Open Cursor or Claude Code in that same folder.
  5. In the AI chat, type: "Install memory-bank for this project and follow the setup instructions."

Reminder: You don't need to understand the code. ✓ It worked if the AI reads memory-bank files and asks you to fill them in. If prompted to sign in, use your T-Mobile GitLab login.

If the command is blocked by your machine: git clone https://gitlab.com/tmobile/ere/memory-bank.git, then copy the contents of its templates/ folder into your project folder.

Make It Global (One-Time)

Copy the rules to your user folder once. Every future project inherits them — no per-project setup beyond the memory-bank scaffold.

Why bother?

  • New projects get security guardrails, code quality, logging, workflow, and Karpathy principles automatically.
  • Slash commands (/feature-dev, /security-review, /code-review) work in every project.
  • You only scaffold memory-bank/ per project — rules are already live.

Where rules live

~/.claude/
  CLAUDE.md            ← Claude Code rules
  AGENTS.md            ← cross-tool rules
  commands/*.md        ← slash commands

~/.cursor/
  rules/*.mdc          ← Cursor rules

Run once per machine

Windows (PowerShell, inside the cloned memory-bank repo)

Copy-Item .\templates\CLAUDE.md "$env:USERPROFILE\.claude\CLAUDE.md"
Copy-Item .\templates\AGENTS.md "$env:USERPROFILE\.claude\AGENTS.md"
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.claude\commands"
Copy-Item .\templates\claude-commands\*.md "$env:USERPROFILE\.claude\commands\"
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.cursor\rules"
Copy-Item .\templates\cursor\rules\*.mdc   "$env:USERPROFILE\.cursor\rules\"

Mac / Linux (Bash)

mkdir -p ~/.claude/commands ~/.cursor/rules
cp ./templates/CLAUDE.md            ~/.claude/CLAUDE.md
cp ./templates/AGENTS.md            ~/.claude/AGENTS.md
cp ./templates/claude-commands/*.md ~/.claude/commands/
cp ./templates/cursor/rules/*.mdc   ~/.cursor/rules/

Restart Claude Code and Cursor so they pick up the new global rules. After that, every project — new or existing — inherits Memory Bank behavior. See docs/GLOBAL-RULES-SETUP.md for organization-wide rollout patterns.
